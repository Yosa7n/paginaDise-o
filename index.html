Diseño de Software: Principios y Mejores Prácticas

Fase crucial en el ciclo de vida del desarrollo de software

¿Qué es el Diseño de Software?

Definición

Proceso de traducir una idea o requisitos del usuario en un plan que captura la funcionalidad necesaria y cómo se construirá.

Alcance

Actividad que sigue a la especificación de requisitos y precede a la programación, conceptualizando y enmarcando sistemas complejos.

Objetivo

Crear una especificación de un artefacto de software diseñado para cumplir objetivos específicos, utilizando componentes primitivos y sujeto a restricciones.

Una planificación intencional y reflexiva durante el diseño de software puede ahorrar tiempo a largo plazo. Un diseño efectivo explora la funcionalidad requerida y ayuda a tomar decisiones sobre cómo cumplirla.

Principios SOLID

SRP

Principio de Responsabilidad Única: Cada clase o módulo debe tener una única responsabilidad y una sola razón para cambiar.

OCP

Principio Abierto-Cerrado: Las entidades de software deben estar abiertas para la extensión pero cerradas para la modificación.

LSP

Principio de Sustitución de Liskov: Una subclase debe poder reemplazar a su clase padre sin romper la funcionalidad esperada.

ISP

Principio de Segregación de Interfaz: Los clientes no deben ser forzados a implementar interfaces que no utilizan.

DIP

Principio de Inversión de Dependencia: Los módulos de alto nivel no deben depender de módulos de bajo nivel; ambos deben depender de abstracciones.

Otros Principios Importantes

YAGNI

You Ain't Gonna Need It: No construir algo innecesariamente y buscar soluciones más simples o reutilizar código existente.

DRY

Don't Repeat Yourself: Promueve la reutilización de lógica y funcionalidad para evitar la duplicación innecesaria.

KISS

Keep It Simple, Stupid: Busca evitar la deuda técnica y los errores reduciendo la complejidad.

Encapsulación

Ayuda a mantener la integridad de los datos al restringir el acceso directo a ciertos detalles de un objeto.

Conceptos Fundamentales

Abstracción

Representar características esenciales sin incluir detalles de fondo.

Modularidad

Dividir la arquitectura del software en componentes llamados módulos.

Arquitectura

Estructura global del software que proporciona integridad conceptual al sistema.

Ocultar información

Diseñar módulos para que la información contenida sea inaccesible para otros módulos que no la necesiten.

Diseño vs Desarrollo de Software

Diseño de Software

  • Ocurre al principio, antes de escribir código
  • Se enfoca en planificación y conceptualización
  • Define la arquitectura y estructura
  • Establece principios y patrones a seguir

Desarrollo de Software

  • Incluye escritura, despliegue y lanzamiento
  • Implementa el diseño previamente establecido
  • Utiliza control de versiones (Git, GitHub)
  • Incluye pruebas unitarias y automatizadas
